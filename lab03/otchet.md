### 2. Инструментарий
Для реализации выбран язык **Python** и следующий стек технологий:
*   **PyQt5**: Для создания современного оконного интерфейса и высокоскоростной отрисовки поля (через буфер `QImage`).

### 3. Описание реализации логики (Клеточный автомат)

Логика моделирования реализована в функции `logic`. Ниже представлен разбор алгоритмов смены состояний.

#### 3.1. Логика воспламенения (Дерево $\to$ Огонь)
Это наиболее сложный блок, учитывающий состояние соседей (окрестность Мура), вектор ветра и рельеф местности.

```python
# Если текущая клетка - Дерево
elif state == S_TREE:
    # Оптимизация: если шанс возгорания 0, пропускаем расчеты
    if p_ignite <= 0.0001:
        next_grid[r, c] = S_TREE
        continue

    catch_fire = False
    
    # 1. Проверка соседей (Окрестность Мура)
    for dr, dc in offsets:
        nr, nc = r + dr, c + dc
        
        if 0 <= nr < rows and 0 <= nc < cols:
            # Если сосед горит
            if grid[nr, nc] == S_FIRE:
                prob = p_ignite # Базовая вероятность
                
                # --- Учет Ветра (Векторное произведение) ---
                # Вектор распространения огня (от соседа к нам)
                vec_x, vec_y = -dc, -dr
                # Скалярное произведение векторов
                # Если ветер совпадает с направлением огня, dot > 0
                dot = (vec_x * wind_x + vec_y * wind_y)
                prob += (dot * wind_str * 0.4)

                # --- Учет Рельефа (Конвекция) ---
                # Разница высот между нами и горящим соседом
                slope = height_map[r, c] - height_map[nr, nc]
                if slope > 0:
                    prob += slope * 6.0 # Огонь идет вверх быстрее
                else:
                    prob -= 0.05        # Вниз идет медленнее
                
                # Итоговый бросок "кубика"
                if np.random.random() < prob:
                    catch_fire = True
                    break
    
    if catch_fire:
        next_grid[r, c] = S_FIRE
```
**Ключевые особенности:**
*   **Ветер:** Рассчитывается через скалярное произведение векторов. Вектор ветра (`wind_x`, `wind_y`) передается из GUI.
*   **Рельеф:** Если текущая клетка выше горящего соседа (`slope > 0`), вероятность возгорания резко возрастает, имитируя физику подъема горячего воздуха вверх по склону.

#### 3.2. Логика горения и затухания (Огонь $\to$ Угли)
Переход от активного пламени к выжженной земле.

```python
# Если клетка - Огонь
elif state == S_FIRE:
    next_grid[r, c] = S_ASH       # Превращаем в угли
    timers[r, c] = 10             # Устанавливаем таймер остывания
```
#### 3.3. Логика остывания почвы (Угли $\to$ Земля)
Моделирование тепловой инерции. На горячей земле лес расти не может.

```python
# Если клетка - Угли (Пепел)
if state == S_ASH:
    if timers[r, c] > 0:
        timers[r, c] -= 1         # Уменьшаем таймер
        next_grid[r, c] = S_ASH   # Остается углем
    else:
        next_grid[r, c] = S_DIRT  # Остыла, стала землей
```
**Пояснение:** Пока таймер не истечет, клетка остается в состоянии `S_ASH`, блокируя рост нового леса.

#### 3.4. Логика роста (Земля $\to$ Лес) и Статика
Восстановление экосистемы и обработка препятствий.

```python
# Рост леса
elif state == S_DIRT:
    if np.random.random() < p_grow:
        next_grid[r, c] = S_TREE

# Статические препятствия
if state == S_ROCK or state == S_WATER:
    next_grid[r, c] = state
    continue
```
**Пояснение:** Скалы и вода являются абсолютными изоляторами. Использование `continue` позволяет исключить их из расчетов, экономя ресурсы процессора.

### 4. Графический интерфейс (GUI)
Разработано приложение на базе `QMainWindow` (библиотека PyQt5).

**Функциональные блоки:**
1.  **Canvas:** Отрисовка поля в реальном времени. Используется прямое преобразование массива NumPy в `QImage`, что позволяет избежать медленной отрисовки отдельных примитивов.
2.  **Панель управления:**
    *   Управление потоком (Старт/Пауза, Новая карта).
    *   Параметры среды (Слайдеры вероятности роста и возгорания).
    *   Вектор ветра (Управление углом 0-360° и силой).
    *   Легенда (Цветовая кодировка состояний).
